# システム全体像

scrum-all-in-one は、スクラム開発を支援するフルスタック Web アプリケーションです。
スプリント管理、チケット管理、デイリースクラム、レトロスペクティブなどの機能を提供します。

### バックエンドの機能モジュール構成

各機能モジュールは以下の 4 ファイルで構成されています（レイヤードアーキテクチャ）:

```
features/<機能名>/
├── routes.ts       # ルートハンドラ（リクエスト受付 & Zod バリデーション）
├── service.ts      # ビジネスロジック
├── repository.ts   # データベースクエリ（Prisma）
└── types.ts        # 型定義
```

```
リクエスト → routes.ts → service.ts → repository.ts → Prisma → PostgreSQL
                ↑                                          ↓
             レスポンス  ←  ←  ←  ←  ←  ←  ←  ←  ←  ←  ←
```

## 4. フロントエンド

### ルーティング構成

| パス | ページ | 認証 | チーム必須 |
|---|---|---|---|
| `/login` | ログイン | 不要 | - |
| `/signup` | サインアップ | 不要 | - |
| `/` | ダッシュボード | 必要 | 必要 |
| `/backlog` | バックログ | 必要 | 必要 |
| `/tickets/:id` | チケット詳細 | 必要 | 必要 |
| `/sprints` | スプリント一覧 | 必要 | 必要 |
| `/sprint` | アクティブスプリント | 必要 | 必要 |
| `/goals` | ゴール管理 | 必要 | 必要 |
| `/dsu` | デイリースクラム | 必要 | 必要 |
| `/working-agreements` | ワーキングアグリーメント | 必要 | 必要 |
| `/retrospectives` | レトロスペクティブ | 必要 | 必要 |
| `/search` | 検索 | 必要 | - |
| `/teams` | チーム管理 | 必要 | - |
| `/users` | ユーザー管理 | 必要（ADMIN） | - |

### 状態管理

| ストア | ライブラリ | 保存先 | 用途 |
|---|---|---|---|
| `useAuthStore` | Zustand | localStorage (`auth-storage`) | 認証状態（トークン、ユーザー情報） |
| `useTeamStore` | Zustand | localStorage | 選択中のチーム ID |
| サーバーステート | TanStack Query | メモリ | API データのキャッシュ & 同期 |

### API クライアント（`lib/api.ts`）

- Axios インスタンスに 2 つのインターセプターを設定:
  1. **リクエスト**: `Authorization: Bearer <token>` と `X-Team-Id` ヘッダーを自動付与
  2. **レスポンス**: 401 エラー時に自動ログアウト（開発用 `dev-token` は除外）
- `VITE_API_URL` 環境変数でバックエンドの URL を指定（未設定時は Vite プロキシ経由）

## 5. バックエンド

### エントリーポイント（`src/index.ts`）

1. 環境変数の読み込み（PaaS では直接注入、それ以外は `.env` ファイル）
2. 本番では必須環境変数（`DATABASE_URL`, `JWT_SECRET`, `JWT_REFRESH_SECRET`）をチェック
3. ミドルウェア適用: logger → secureHeaders → CORS → errorHandler
4. ルート設定 → HTTP サーバー起動（デフォルト :4000）

### API エンドポイント一覧

#### 認証（ミドルウェアなし）

| メソッド | パス | 説明 |
|---|---|---|
| POST | `/auth/signup` | ユーザー登録 |
| POST | `/auth/login` | ログイン |
| POST | `/auth/refresh` | トークンリフレッシュ |
| POST | `/auth/logout` | ログアウト |

#### API（認証必須）

| メソッド | パス | チームスコープ | 説明 |
|---|---|---|---|
| GET/POST | `/api/tickets/*` | 必要 | チケット CRUD、ステータス変更、コメント、DoD |
| GET/POST | `/api/sprints/*` | 必要 | スプリント CRUD、開始/完了 |
| GET/POST | `/api/goals/*` | 必要 | 長期ゴール & スプリントゴール |
| GET/POST | `/api/dsu/*` | 必要 | デイリースクラムログ |
| GET | `/api/dashboard/*` | 必要 | ダッシュボードデータ |
| GET/POST | `/api/working-agreements/*` | 必要 | ワーキングアグリーメント |
| GET/POST | `/api/retrospectives/*` | 必要 | レトロスペクティブ |
| GET | `/api/search` | 不要 | 横断検索 |
| GET/POST | `/api/teams/*` | 不要 | チーム管理 |
| GET/PUT | `/api/users/*` | 不要 | ユーザー管理 |

### ミドルウェア

| ミドルウェア | 適用範囲 | 役割 |
|---|---|---|
| `logger` | 全リクエスト | リクエストログ |
| `secureHeaders` | 全リクエスト | セキュリティヘッダー付与 |
| `cors` | 全リクエスト | CORS 制御（`FRONTEND_URL` のみ許可） |
| `errorMiddleware` | 全リクエスト | エラーハンドリング |
| `authMiddleware` | `/api/*` | JWT 検証 & ユーザー情報設定 |
| `teamMemberGuard` | チームスコープのルート | `X-Team-Id` ヘッダー検証 & チームメンバーチェック |

## 7. 認証フロー

### トークン戦略

| トークン | 署名鍵 | 有効期限 | ペイロード |
|---|---|---|---|
| アクセストークン | `JWT_SECRET` | 15 分 | `sub`(ユーザーID), `email`, `role`, `organizationId` |
| リフレッシュトークン | `JWT_REFRESH_SECRET` | 7 日 | `sub`(ユーザーID), `email` |

### 認証フロー図

```
1. ログイン / サインアップ
   ブラウザ → POST /auth/login → バックエンド
   ブラウザ ← { accessToken, refreshToken, user } ← バックエンド

2. API リクエスト（通常）
   ブラウザ → Authorization: Bearer <accessToken> → バックエンド
   バックエンド: JWT 検証 → ユーザー情報をコンテキストに設定

3. トークンリフレッシュ（アクセストークン期限切れ時）
   ブラウザ → POST /auth/refresh { refreshToken } → バックエンド
   ブラウザ ← { accessToken, refreshToken } ← バックエンド

4. 401 エラー時
   ブラウザ: 自動ログアウト → /login にリダイレクト
```

### 開発環境の便利機能

- `NODE_ENV !== 'production'` の場合、`dev-token` を使うと認証をバイパスできる
- 開発用モックデータへのフォールバックあり

## 8. 機能一覧

| 機能 | 説明 |
|---|---|
| **ダッシュボード** | スプリント進捗、ベロシティ、チケット統計の可視化 |
| **チケット管理** | USER_STORY / TASK / BUG / SUBTASK の CRUD、ステータス管理、親子関係、コメント |
| **バックログ** | 未割当チケットの管理、優先度順のソート |
| **スプリント管理** | スプリントの作成・開始・完了、バーンダウンチャート |
| **ゴール管理** | 長期ゴール（PI）とスプリントゴールの階層管理 |
| **デイリースクラム（DSU）** | 各メンバーの昨日・今日・ブロッカーを記録 |
| **レトロスペクティブ** | KPT 等のフォーマット、カード/ボードモード、投票、アクションアイテム |
| **ワーキングアグリーメント** | チームのルール管理、カテゴリ分類、変更履歴 |
| **DoD（Definition of Done）** | チケットごとの完了基準チェックリスト |
| **キャパシティプランニング** | スプリントごとのメンバー稼働可能日数管理 |
| **チームカレンダー** | メンバーの休暇・不在管理 |
| **横断検索** | チケット・ゴール等の全文検索 |
| **チーム管理** | チーム作成、メンバー招待、ロール割当 |
| **ユーザー管理** | ユーザー一覧、ロール変更（ADMIN のみ） |
| **ベロシティトラッキング** | ストーリーポイント / チケット数の切り替え対応 |
